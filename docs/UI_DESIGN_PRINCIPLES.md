# UI Design Principles

This document captures general UI preferences for this project.

## Visual Style

- Prefer flat UI. Avoid shadows on cards, blocks, or popovers unless strictly necessary.
- Borders should be subtle (soft gray). Avoid high-contrast black borders that pull attention away from content.
- Keep chrome lightweight so content is the visual focus.

## Layout & Density

- Use consistent spacing and predictable alignment. Avoid heavy framing or visually noisy separators.
- Choose minimal structure over decorative layout elements.

## Tables & Lists

- Avoid unnecessary borders. For example, in task tables, do not add borders unless they improve readability.
- Prefer subtle row separators or spacing over strong grid lines.

## Interaction

- Use clear, restrained affordances (hover states, muted outlines) instead of heavy visual effects.
- Keep controls discoverable but not visually dominant.
